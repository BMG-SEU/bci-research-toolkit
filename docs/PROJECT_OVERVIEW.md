# BCI Research Toolkit — 项目介绍

> 一套为脑机接口与脑磁/脑电信号研究打造的 AI 智能体工具包

---

## 一、这是什么？

**BCI Research Toolkit** 是一个让多个 AI 专家自动协作完成科研任务的工具包。它不是软件、不是 App、不是一个网页——而是一套配置文件（YAML + Markdown），装在你的电脑上，通过命令行或浏览器使用。

### 一句话理解

你对着电脑说：「帮我写一段论文 Discussion」，背后会有论文写作专家、文献综述专家、统计方法专家同时干活，自动协作、相互传结果，最后汇总成一篇完整的 Discussion 交给你。

---

## 二、解决什么问题？

课题组日常科研中的痛点：

| 痛点 | 本工具解决方案 |
|------|--------------|
| MEG/EEG 预处理参数不知道怎么选 | AI 自动评估数据特征，推荐滤波/ICA/分段参数 |
| 文献太多，不知道怎么系统化整理 | AI 自动设计检索策略、分类文献、生成综述 |
| 论文 Discussion 写不出来 | 多角色分析结果 → 对比文献 → 讨论局限 → 生成完整 Draft |
| 英文表达不地道 | 三层次润色（语法→表达→学术惯例），期刊风格化 |
| 数据处理和分析代码不会写 | 自动生成 MNE-Python 代码框架 |
| 审稿意见不知道怎么回复 | 逐条分解、分类处理、自动写回复信 |
| 课题组知识传承困难 | 工作流模板可复用，一次写好，全组受益 |

## 三、项目有什么？

### 内容清单

| 组件 | 数量 | 格式 | 说明 |
|------|:--:|------|------|
| 科研 Agent | 7 个 | Markdown | AI 角色定义（MEG分析师、EEG预处理专家、论文写手等） |
| 科研 Skills | 23 个 | Markdown | 工作方法论（论文写作、润色、文献搜索、时频分析等） |
| 预设 Workflow | 11 个 | YAML | 可复用的多 Agent 协作流程模板 |
| 提示词库 | 89+ 个 | Markdown | 来自 academic-ai-prompt-main 的高品质学术提示词 |
| 部署脚本 | 2 个 | Shell/Batch | 一键安装所有依赖 |
| Web 界面 | 1 个 | — | AO 自带，浏览器操作 |
| Docker 镜像 | 1 个 | Dockerfile | 零环境依赖部署 |
| CI/CD | 1 个 | GitHub Actions | 自动校验 YAML |
| 文档 | 5 份 | Markdown | 任务书、教程、FAQ、测试计划、测试报告 |

### 7 个科研 Agent

| Agent | 专长 | 典型任务 |
|-------|------|---------|
| MEG 分析专家 | MEG 数据处理、源定位、功能连接 | 预处理方案、时频参数建议 |
| EEG 预处理专家 | EEG 滤波、ICA、伪影去除 | 去眼电/肌电策略、坏段检测 |
| BCI 文献综述员 | 系统化文献检索、分类、评估 | 检索策略、综述框架 |
| 论文写作专家 | IMRaD 写作、期刊风格适配 | 各章节撰写、摘要生成 |
| 统计方法顾问 | 实验设计、假设检验、效应量 | 统计方法选型、结果审核 |
| 图表设计师 | 科学可视化、matplotlib+MNE | 配色方案、图表代码 |
| 代码审查员 | MNE-Python 最佳实践 | 代码正确性/性能/可复现性 |

### 23 个科研 Skills（按来源分类）

**Nature 系列（8 个）**：论文写作 · 润色 · 文献搜索 · 引用管理 · 数据分析 · 图表生成 · 论文精读 · 审稿回复

**PaperSpine 系列（10 个）**：人性化改写 · 中英互译 · 引用银行 · 论文重写 · 期刊调研 · 论文组装 · LaTeX排版 · 输出审计 · 工作流配置 · 工具包更新

**EEG/MEG 专用（5 个）**：EEG去伪影 · 时频分析 · 统计检验 · 源定位 · BCI机器学习

### 11 个预设 Workflow

| Workflow | 输入 | 产出 |
|----------|------|------|
| MEG 预处理 | MEG 文件路径 | 数据报告 + 预处理方案 + Python代码 |
| EEG 分析 | EEG 文件 + 分析类型 | 完整分析 Pipeline + 代码 |
| 文献综述 | 研究主题 | 综述初稿（含检索策略+分类+对比+未来方向） |
| 论文方法学 | 实验设计参数 | 完整 Methods 章节 |
| 论文结果 | 数据摘要 | 完整 Results 章节 |
| 论文讨论 | 结果 + 假设 + 局限 | 完整 Discussion 章节 |
| 论文摘要 | 背景/方法/结果/结论 | 格式化摘要 + 关键词 |
| 图表生成 | 数据描述 | 三步法：分析→选图→Python代码 |
| 源定位 | MEG/EEG 文件 | 源定位方案 + MNE代码 |
| 功能连接 | MEG/EEG 文件 | 连接分析方案 + 代码 |
| 数据读取桥接 | 真实 .fif 文件 | AO规划 + Claude Code读文件 + 分析报告 |

## 四、架构设计

### 整体架构

```
┌─────────────────────────────────────────────┐
│                 课题组成员                     │
│         浏览器 (ao web)  │  命令行            │
└──────────────┬──────────────┬───────────────┘
               │              │
┌──────────────┴──────────────┴───────────────┐
│          Agency Orchestrator (AO)            │
│            多 Agent 编排引擎                   │
│   · 读 YAML → 构建 DAG → 并行调度             │
│   · 加载 Agent 角色 → 注入 Skills 方法论      │
│   · 步间变量传递 → 失败重试 → 汇总输出         │
└──────────────┬──────────────────────────────┘
               │
     ┌─────────┼──────────┬──────────────┐
     ▼         ▼          ▼              ▼
┌─────────┐ ┌──────┐ ┌─────────┐ ┌───────────┐
│DeepSeek │ │科研   │ │科研     │ │Claude Code│
│  GLM    │ │Skills │ │Agents   │ │(可选)     │
│  API    │ │23个   │ │7个      │ │写码/调试   │
└─────────┘ └──────┘ └─────────┘ └───────────┘
```

### 技术选型

| 组件 | 选型 | 为什么 |
|------|------|--------|
| 编排引擎 | agency-orchestrator | 零代码 YAML + 自然语言编排 + DAG 自动并行 |
| 主力 LLM | DeepSeek API | ¥2/1M tokens，论文 Discussion ¥0.16 |
| 备用 LLM | GLM API | 高校常有免费额度 |
| 代码执行 | Claude Code | 读写文件 + 跑 Python + 调试 |
| 方法论 | superpowers-zh | 20 个通用开发方法论 |
| 运行时 | Node.js ≥ 18 | AO 运行时 |
| 容器化 | Docker | 零环境依赖、服务器部署 |
| CI/CD | GitHub Actions | 自动校验 YAML |

### 核心理念：AO 管「想」，Claude Code 管「做」

```
你: "帮我处理 MEG 数据"

AO（项目经理）:                   Claude Code（程序员）:
  拆任务 → 配角色 → 并行走          不需要多角色协作时单干
  多视角碰撞 → 写方案 → 出报告       直接操作文件 → 跑代码 → 调试
  适合：分析、规划、写作            适合：实现、调试、重构
  
  90% 日常科研用它                  10% 写代码调试用它
```

## 七、典型使用流程

```
上午：
  ao run workflows/literature-review.yaml -i topic="deep learning motor imagery"
  → 产出：综述初稿（含检索策略、分类、方法回顾）

下午：
  ao run workflows/paper-methods.yaml -i design="..." -i participants="..." ...
  → 产出：完整 Methods 章节

  ao run workflows/paper-results.yaml -i neural="alpha ERD显著" ...
  → 产出：完整 Results 章节

晚上：
  ao run workflows/paper-discussion.yaml -i results_summary=@results.md ...
  → 产出：完整 Discussion 章节

明天：
  ao compose "帮我润色整篇论文英文" --run
  → 产出：润色后的论文
```
